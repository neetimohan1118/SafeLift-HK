# SafeLift HK — 工地安全智能助手

### 項目概覽 ｜ 技術與產品介紹

---

## 一、項目簡介

**SafeLift HK** 是一套專為香港建造業設計的**智能工地安全管理系統**，以吊運作業為切入點，結合 **AI 影像分析**技術，協助工地人員即時識別現場危害、管理設備資產、以及數碼化管理各類安全文件。

系統以 **Progressive Web App (PWA)** 形式運行，用戶無需安裝即可透過手機瀏覽器使用全部功能，後續將擴展至原生手機應用。

---

## 二、要解決的問題

香港建造業長期面對以下痛點：

**工地安全層面：**
根據香港勞工處資料，建造業一直是職業傷亡的高危行業。吊運作業涉及重型機械與高空作業，是工地意外的主要來源之一。目前工地的危害識別主要依賴人手巡查，容易受到經驗、疲勞和天氣等因素影響，存在明顯的識別盲區。

**設備管理層面：**
貨車吊機等設備需要按照《工廠及工業經營（起重機械及起重裝置）規例》（Cap. 59J）定期進行法定檢驗。但很多中小型承建商仍以紙本方式管理證書和維修紀錄，導致證書到期未續、維修紀錄散失等問題。

**文件管理層面：**
安全證書、檢驗報告、維修單據等文件分散在不同地方，當勞工處進行突擊檢查時，往往難以即時出示所需文件。

## 三、產品三大核心模組

### 模組一：AI 現場危害識別

用戶在工地打開系統，用手機相機拍攝吊運現場。系統透過 **AI 視覺分析**自動識別以下情況：

| 偵測項目 | 說明 |
|---------|------|
| 人員進入危險範圍 | 偵測吊運範圍內是否有人逗留 |
| 不當吊索用法 | 吊索打結、夾角過大、缺少安全扣等 |
| 貨物偏斜或超出範圍 | 貨物重心偏移、超出吊運半徑 |
| 個人防護裝備缺失 | 安全帽、反光衣等是否佩戴 |
| 地面穩定性 | 吊機是否放置在不穩定地面 |

**用戶體驗流程：**

```
拍攝相片 → AI 自動分析 (3–5秒) → 顯示危害清單及嚴重等級
                                         │
                                    用戶確認危害
                                         │
                                 自動生成危害紀錄
                              （日期 + GPS位置 + 相片）
```

系統會以顏色標示危害等級：

- 🔴 **嚴重 (Critical)** — 需要立即停工處理
- 🟠 **高危 (High)** — 需要盡快糾正
- 🟡 **中等 (Medium)** — 需要留意及跟進
- 🔵 **低危 (Low)** — 建議改善

**技術亮點：**
- 採用**全中國 AI 多模型架構**，以阿里巴巴 **Qwen2.5-VL** 為主力，智譜 **GLM-4.6V** 及字節跳動 **豆包 Vision** 為備用
- Qwen2-VL 在工地安全偵測基準測試中達到 **89.4% 精確度**，為目前公開數據中最高
- 每張相片分析成本僅約 **HK$0.025**，極具成本效益
- 所有模型均採用 OpenAI 兼容 API 格式，可靈活切換，無供應商綁定風險
- 支援離線拍攝，恢復連線後自動上傳分析

---

### 模組二：設備資產管理

每部吊機、每套起重裝置在系統中都有一個完整的「數碼檔案」：

| 資料項目 | 內容 |
|---------|------|
| 基本資料 | 車牌號碼、設備編號、型號、製造商 |
| 性能參數 | 最大承載能力（噸）、最大工作半徑（米） |
| 證書狀態 | 法定檢驗證書到期日、剩餘天數、狀態指示 |
| 維修紀錄 | 最近維修保養日期、歷史維修記錄 |
| 關聯文件 | 所有相關證書、檢驗報告、維修單據 |
| 危害紀錄 | 該設備歷來的危害報告 |

**核心功能：**

- **QR Code 掃描** — 每部設備貼上專屬 QR Code，工地人員掃一掃即可查看所有資料
- **到期提醒系統** — 證書到期前 30 日及 14 日自動發送提醒（應用內通知 + 電郵）
- **狀態一目了然** — 綠色（有效）、黃色（即將到期）、紅色（已過期）

---

### 模組三：文件上載與管理

| 功能 | 說明 |
|-----|------|
| 文件上載 | 支援 PDF 及照片格式的證書、檢驗報告、維修單據 |
| 設備連結 | 將文件「連結」到指定設備，建立完整檔案 |
| 智能搜尋 | 輸入車牌號碼或設備編號，即時查看所有相關資料 |
| 文件分類 | 證書、檢驗報告、維修紀錄、危害報告自動分類 |
| 到期追蹤 | 證書類文件自動追蹤有效期 |

**應用場景：** 當勞工處進行現場檢查時，工地人員只需在手機上掃描設備 QR Code 或輸入車牌號碼，即可在數秒內出示所有法定文件。

## 四、技術架構

### 整體架構圖

```
┌─────────────────────────────────────────┐
│           用戶端 (PWA 網頁應用)            │
│                                         │
│   相機拍攝  ←→  QR掃描  ←→  文件上載      │
│              ↓                           │
│        離線快取層 (Service Worker)         │
└──────────────┬──────────────────────────┘
               │
     ┌─────────┴─────────┐
     │   Firebase 雲端服務  │
     │                     │
     │  Firestore 資料庫    │  ← 設備、危害、文件資料
     │  Cloud Storage      │  ← 相片、PDF 儲存
     │  Cloud Functions    │  ← 後端邏輯處理
     │  Authentication     │  ← 用戶驗證
     └─────────┬──────────┘
               │
     ┌─────────┴─────────┐
     │   AI 多模型路由器    │
     │                     │
     │  主力: Qwen2.5-VL   │  ← 阿里巴巴
     │  備用: GLM-4.6V     │  ← 智譜AI
     │  輕量: 豆包 Vision   │  ← 字節跳動
     └────────────────────┘
```

### 技術選型

| 層級 | 技術 | 選擇原因 |
|-----|------|---------|
| 前端框架 | Next.js 14 + React | 業界主流、支援伺服器端渲染、PWA 插件豐富 |
| UI 組件 | Tailwind CSS + shadcn/ui | 快速開發、一致的設計系統 |
| 資料庫 | Firebase Firestore | 內建離線同步、實時更新、亞洲區域支援 |
| 文件儲存 | Firebase Cloud Storage | 安全可靠、自動擴展 |
| AI 引擎（主力） | Qwen2.5-VL（阿里巴巴） | 工地安全偵測精確度 89.4%、成本極低、開源可微調 |
| AI 引擎（備用） | GLM-4.6V（智譜AI） | 免費 Flash 版本可用、128K 上下文、自動故障轉移 |
| AI 引擎（輕量） | 豆包 Vision（字節跳動） | 最低成本方案、每張僅 HK$0.008 |
| 離線支援 | Workbox + Firestore | Service Worker 快取 + IndexedDB 自動同步 |
| 多語言 | next-intl | 繁體中文 / English 雙語支援 |

### 為什麼選擇 PWA？

| 優勢 | 說明 |
|-----|------|
| 跨平台 | 一次開發，iOS、Android、桌面電腦全部適用 |
| 無需安裝 | 用戶透過瀏覽器即可使用，也可「安裝」到主屏幕 |
| 離線運作 | 工地網絡環境差也能拍照和查看資料，恢復連線後自動同步 |
| 快速迭代 | 更新即時生效，無需等待 App Store 審核 |
| 低門檻 | 前線工人無需下載 App，掃碼即用 |
| 未來擴展 | 可透過 Capacitor 封裝為原生 App 上架 App Store / Play Store |

## 五、AI 技術能力展示

### 分析流程

```
用戶拍攝工地相片
       │
       ▼
客戶端壓縮至 1.5MB 以下
       │
       ▼
上傳至 Cloud Functions
       │
       ▼
AI 多模型分析（全中國 AI）
（主力 Qwen2.5-VL → 備用 GLM-4.6V → 豆包 Vision）
（專門調校的工地安全分析提示詞）
       │
       ▼
返回結構化 JSON 結果
 ├── 偵測到的危害清單
 ├── 每項危害的嚴重等級
 ├── 中英文描述
 ├── 信心指數 (0-1)
 └── 建議糾正措施
       │
       ▼
用戶在介面上查看結果
確認 / 忽略 / 補充說明
```

### AI 偵測能力範圍

| 類別 | 偵測內容 | 信心門檻 |
|-----|---------|---------|
| 人員安全 | 吊運範圍內有人逗留、缺乏安全距離 | ≥ 70% |
| 吊索用法 | 打結、夾角過大(>120°)、缺少安全扣 | ≥ 70% |
| 貨物狀況 | 偏斜、超出範圍、不平衡 | ≥ 70% |
| 個人裝備 | 缺少安全帽、反光衣 | ≥ 70% |
| 地面環境 | 不穩定地面、障礙物 | ≥ 70% |

- 信心指數 70%–85% 的結果會標記為「需要人手確認」
- 低於 70% 的結果自動過濾，避免誤報
- 系統持續收集用戶回饋，用以優化分析準確度

## 六、合規性 — 香港法例要求

系統設計嚴格對應香港《工廠及工業經營條例》相關規例：

| 法規要求 | 系統功能 |
|---------|---------|
| Cap. 59J — 起重機年度檢驗 | 自動追蹤檢驗證書到期日，提前 30 日提醒 |
| 表格二 (Form 2) — 錨固/壓載檢驗 | 文件上載及連結至設備 |
| 表格三 (Form 3) — 起重機檢驗證書 | 證書數碼化管理，即時查閱 |
| 起重裝置 — 每 6 個月檢驗 | 吊索、鏈條、鈎具獨立追蹤週期 |
| 操作員證書 | 操作員資格證書紀錄 |
| 現場文件保存 | 所有法定文件可透過 QR Code 即時調出 |

## 九、產品路線圖

### 第一階段：MVP（0–3 個月）

- 三大核心模組上線（危害識別、設備管理、文件管理）
- PWA 網頁版本，支援 iOS 及 Android 瀏覽器
- 全中國 AI 多模型影像分析（Qwen2.5-VL + GLM-4.6V + 豆包 Vision）
- 離線拍攝及同步功能
- QR Code 掃描設備
- 中英雙語介面

### 第二階段：功能強化（4–6 個月）

- 以收集的工地數據微調 Qwen2-VL 開源模型，進一步提升偵測準確度
- 自建模型伺服器，大幅降低 AI 分析成本
- 自訂預製檢查清單（安全巡查、開工前檢查）
- 團隊管理及多項目支援
- 進階數據分析儀表板（危害趨勢、設備使用率）
- PDF 報告匯出（公司品牌定制）

### 第三階段：手機應用（6–9 個月）

- 透過 Capacitor 封裝為原生 iOS / Android App
- App Store / Google Play 上架
- 原生推送通知
- NFC 設備標籤支援
- 生物識別登入（Face ID / 指紋）

### 第四階段：企業版（9–12 個月）

- 多租戶企業架構
- 與現有 ERP / 安全管理系統整合 API
- 預測性維護（AI 分析維修趨勢）
- 法規自動更新
- 完整審計追蹤與合規報告

## 十一、為什麼是現在？

**行業趨勢推動：**

- 香港政府積極推動建造業數碼化（建造業議會 CIC 的 Construction 2.0 策略）
- AI 視覺分析技術在 2024–2025 年達到實用水平，成本大幅下降
- 建造業安全法規執法趨嚴，企業合規壓力增加
- 大型基建項目持續推出，行業對安全管理工具的需求不斷增長
- 新一代前線工人對數碼工具的接受度提高

**技術成熟度：**

- 大型語言模型的視覺能力已能勝任基本的工地危害識別
- 中國 AI 模型（阿里巴巴 Qwen、智譜 GLM 等）在視覺分析領域表現出色，且對香港用戶無地域限制
- 開源模型（Qwen2-VL）允許企業自行微調和部署，降低長期成本並提升準確度
- PWA 技術允許免安裝、離線使用，完美配合工地環境
- 雲端服務成本持續下降，讓中小企也能負擔 AI 工具
